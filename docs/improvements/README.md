# Improvements and Future Extensions

This folder keeps optional improvements separate from the core Phase 1 implementation so you can modify, accept, or reject them later without mixing them into setup docs.

## Files

- `phase-1-hardening.md`: near-term improvements before calling Phase 1 production-ready.
- `memory-rag-plan.md`: future memory and RAG-based memory design.
- `tools-extension-guide.md`: how to add new tools without changing the pipeline.
- `email-integration-plan.md`: future email sending and reading integration plan.
- `guest-mode-plan.md`: options for real guest mode beyond the current frontend-only preview.
